# 📁 Peer‑to‑Peer Folder Sharing (без хранения файлов на сервере)

> **Безопасный совместный доступ к папкам и файлам** с разных платформ.
> Сервер хранит **только метаданные** – сами файлы остаются у пользователей.

---

## 🎯 Цель

Обеспечить пользователям удобный и защищённый общий доступ к своим папкам и файлам через Web, Windows и Android без загрузки файлов в облако.

---

## 🚀 Ключевые возможности

- 🔐 Регистрация и вход по email/паролю (OAuth 2.0)
- 📂 Создание, просмотр, загрузка и удаление **своих** папок и файлов (хранятся только метаданные)
- 👥 Назначение прав доступа другим пользователям (`просмотр` / `полный доступ`)
- 🔗 Генерация временных ссылок для незарегистрированных пользователей
- 📁 Отдельная вкладка «Доступные папки» с фильтрацией по владельцу, названию, правам
- 🔍 Расширенный поиск файлов и папок
- 🪟 Интеграция с **Проводником Windows** (виртуальная папка)
- 📧 Уведомления на email и внутри приложения
- 🛡️ Все действия через **HTTPS**, проверка прав на каждом запросе
- 🧑‍💼 Административная панель (просмотр метаданных, управление пользователями и правами)

---

## 🧱 Архитектура (кратко)

- **Backend**: центральный REST API (ASP.NET Core)
- **База данных**: PostgreSQL (только метаданные)
- **Frontend**: React (Web), Android (Kotlin/Java), Windows (C# + Shell Extensions)
- **Аутентификация**: OAuth 2.0 (password flow)
- **Уведомления**: email (SMTP) + внутренние push через webhooks
- **Безопасность**: HTTPS, хэширование токенов, проверка доступа на каждом запросе

Файлы **не хранятся** на сервере – клиенты работают напрямую с локальными данными, сервер управляет только правами и метаинформацией.

---

## 📦 Основные компоненты (репозитории)

- `backend-rest-api` – центральный REST API
- `web-client-react` – веб‑интерфейс
- `android-client` – Android‑приложение
- `windows-client` – Windows‑клиент + интеграция с Проводником
- `db-migrations` – миграции PostgreSQL
